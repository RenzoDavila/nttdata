Explicación

Se crea un módulo nuevo con el archivo "spec".

Agregamos un mock del servicio (mockedPrincipalService) y creamos un objeto (UserPruebasUnitariasData) que va a mostar la respuesta esperada

Colocamos un título relacionado con las pruebas que vamos a hacer, y definimos el uso del mock en los providers.

Utilizamos la sintaxis "it" para definir las pruebas a realizar en esto dos casos que retorne un objeto igual a "UserPruebasUnitariasData" y que al existir un error el array "user" se mantenga vacio.

El ejecutar "ng test" podemos ver que las dos pruebas realizadas a nuestro servicio retornaron los resultados esperados.
