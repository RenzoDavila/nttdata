Estrategias Prácticas de Optimización

Build para producción

Al colocar el comando "ng build --configuration production" indicamos a Angular que se optimice para que el tamaño de los archivos al compilador sea mucho menor y por lo tanto la carga será más rápida, a su vez si agregamos "--aot" indicamos que revise el código para optimizarlo.

Refactorización de componentes y funciones

Algo completamente necesario en Angular es crear módulos/componentes que nos permitan reutilizar las vistas y funciones a lo largo de la aplicación, esto hace que nuestro código disminuya notablemente y permite que el mantenimiento de la aplicación sea mil veces más sencillo.

Tubería asíncrona

Cuando hacemos uso de observables, al no hacer uso de "unsubscribe" podemos provocar pérdidas de memoria, pero con el uso de "pipe async" esto se simplifica dado de que el mismo se suscribe y se des-suscribe de los servicios automáticamente y como un extra tenemos un código más limpio
